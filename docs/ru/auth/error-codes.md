# Справка по кодам ответов API

## HTTP статусы

| Статус | Описание |
|:------:|----------|
| 200, 201, 202 | Успешное выполнение операции |
| 400 | Ошибка в запросе |
| 401 | Недостаточно прав для выполнения операции |
| 403 | Доступ запрещён |
| 404 | Неверный URL |
| 500 | Внутренняя ошибка сервера |
| 503 | Превышен лимит запросов (600 req/min) |

## Коды ошибок (error_code)

| Код | Описание |
|:---:|----------|
| 0 | Срок действия API ключа истёк |
| 1 | Токен не существует / недействителен |
| 2 | Недостаточно прав для выполнения операции |
| 3 | Неверный формат запроса |
| 4 | Превышен лимит запросов |
| 5 | Refresh token не найден |
| 6 | Превышен лимит токенов |

## Формат ответа с ошибкой

```json
{
  "error": "invalid_token",
  "error_description": "Token doesn't exist",
  "error_code": 1
}
```

| Поле | Описание |
|------|----------|
| `error` | Тип ошибки |
| `error_description` | Человекочитаемое описание |
| `error_code` | Числовой код ошибки |

## Лимит запросов

Максимум **600 запросов в минуту** на одно приложение. При превышении возвращается статус 503.
