# Введение

Admitad использует открытый протокол [OAuth 2.0](http://tools.ietf.org/html/draft-ietf-oauth-v2-16/) для авторизации приложений.

## Типы авторизации

- [Авторизация стороннего web-приложения](./authorization-code.md)
- [Клиентская авторизация](./client-credentials.md)
- [Обновление access_token](./refresh-token.md)
- [Описание строки signed_request](./signed-request.md)
- [Справка по настройкам доступа](./scopes.md)
- [Справка по кодам ответов API](./error-codes.md)

## Процесс авторизации

После успешного прохождения авторизации выдается ключ доступа `access_token`, с помощью которого вы можете выполнять запросы к Admitad API.

Доступность методов API зависит от набора полученных прав доступа (scopes) — некоторые функции могут быть ограничены в зависимости от выданных разрешений.
